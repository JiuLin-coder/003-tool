.gitignore 的生效时机

无需先提交 .gitignore 文件：只要在项目根目录（或子目录）创建了 .gitignore 文件，Git 会立即根据其规则忽略匹配的文件。即使该文件尚未提交到仓库，规则也会在本地生效。
每次操作均自动扫描：当执行 git add、git commit 等命令时，Git 会实时扫描 .gitignore 文件，并自动排除符合规则的文件。

.gitignore 文件应包含不需要提交到 Git 仓库的文件类型，例如：

1. 操作系统生成的文件，如 macOS 的 .DS_Store。
2. 代码编辑器和集成开发环境（IDE）生成的配置文件。
3. 编程语言或框架自动生成的文件，如编译后的 .o 文件。
4. 软件包管理器生成的文件夹，如 npm 的 node_modules。
5. 包含敏感数据和个人信息的文件，如 .env 文件。
6. 运行时文件，如 .log 文件。

在 Git 中忽略文件和目录
要忽略特定文件或目录，只需在 .gitignore 文件中指定它们的路径。例如，

1. 要忽略根目录下的 text.txt 文件，可以添加以下内容：/text.txt
2. 忽略特定目录及其内容，需要在目录名后加上斜线 /：test/
3. 如果要忽略所有具有特定名称的文件，无需指定完整路径，只需写出文件名：text.txt
4. 忽略以特定单词开头或结尾的文件或目录，可以使用 _ 通配符：
   img\_ # 忽略所有以 img 开头的文件和目录
   _.md # 忽略所有以 .md 结尾的文件
5. 如果需要对某些规则进行例外处理，可以使用 ! 来排除特定文件：
   \*.md #忽略所有 .md 文件
   !README.md #但不忽略 README.md 文件
6. 忽略，曾经被提交过的（或已经被add），要先git rm --cached <文件名>，然后.gitignore再写入文件名，然后再git add . git commit
